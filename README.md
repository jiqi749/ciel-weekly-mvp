# Ciel · 一周怎么过

一个纯静态、local-first 的个人时间与注意力管理 MVP。

它不要求记录全天流水，核心是：

- 每周先保护 3 段作品集产出时间；
- 提前安排并确认认真陪伴和攀岩时间；
- 临时 Todo 先进入收件箱，不自动挤进今天；
- 作品集完成时留下代码、文档或问题解决证据；
- 加班等意外发生后重新安排，不用睡眠填补冲突；
- 出差、高压或恢复期可以切换为特殊周。
- 使用同一邮箱和同步密码登录后，在手机和电脑之间自动同步。

## 数据

数据始终先保存在当前浏览器的 `localStorage`。登录后，整份状态会同步到 Supabase；断网时仍然可以继续使用，恢复网络后自动上传。右上角“备份”可以额外导出 JSON 文件。

同步数据通过 Row Level Security 限定为当前登录用户。首次登录时，如果本机和云端都有内容，系统会要求选择保留哪一份，不会静默覆盖。

## 同步配置

- 公开连接信息位于 `sync-config.js`；
- 数据表与 RLS 规则位于 `supabase-schema.sql`；
- Supabase Auth 的站点地址和 Redirect URL 均指向 GitHub Pages 地址。
- Supabase Auth 的 `Confirm email` 已关闭，让单人 MVP 不依赖默认邮件投递。

## 运行

这是一个无需构建的静态网站，入口文件为 `index.html`，可部署到任意静态托管服务。
